# Contributing to TCA

Thank you for your interest in contributing to Terminal Colors Architecture!

This repository contains the **TCA specification** and the **tca-cli** reference tool. Themes are maintained by [Tinted Theming](https://github.com/tinted-theming/home). If you want to contribute a theme, do so there.

## Contributing to the Specification

The specification lives in [`SPECIFICATION.md`](SPECIFICATION.md).

### Process

1. Open an issue to discuss the change.
2. Fork this repository and create a branch
3. Edit `SPECIFICATION.md`
4. Submit a pull request with a clear description of what changed and why

### Guidelines

- Follow [RFC 2119](https://soundcloud.com/ericwbailey/rfc-2119) keyword conventions (MUST, SHOULD, MAY)
- Keep normative requirements (MUST/MUST NOT) minimal, only mandate what is necessary for interoperability
- Changes that affect library conformance or the semantic mapping tables should note the impact on existing implementations

## Contributing to tca-cli

The reference CLI tool lives in [tca-rust](https://github.com/carmiac/tca-rust). File issues and PRs there.

## Questions?

Open an issue for discussion.

## Code of Conduct

Be respectful and constructive.
